Day 2 – Data Preprocessing and Exploratory Data Analysis

Overview

On the second day of my internship, I focused on understanding the data preprocessing pipeline and the importance of preparing data before training machine learning models. I worked with different datasets using Python and explored various techniques to clean, analyze, and transform data.

Learning Objectives

• Learn how to load datasets using different Python libraries.
• Understand the importance of data cleaning.
• Perform Exploratory Data Analysis (EDA).
• Encode categorical variables.
• Perform feature engineering.
• Handle imbalanced datasets.

Topics Covered

Dataset Loading

• Loaded datasets using Pandas.
• Worked with datasets available in Seaborn.
• Loaded built-in datasets from Scikit-learn.

Data Cleaning

• Identified missing values.
• Removed duplicate records.
• Filled missing numerical values using the median.
• Filled missing categorical values using the mode.

Exploratory Data Analysis (EDA)

• Explored the structure of the dataset.
• Generated descriptive statistics.
• Visualized data using histograms, box plots, and correlation heatmaps.
• Understood relationships between different features.

Encoding

• Learned Label Encoding.
• Applied One-Hot Encoding to categorical variables.
• Converted categorical values into numerical format for machine learning models.

Feature Engineering

• Created new features from existing data.
• Converted date columns into datetime format.
• Extracted useful information from existing columns.
• Applied transformations to improve data quality.

Handling Imbalanced Data

• Learned about class imbalance.
• Understood why balanced datasets are important.
• Used SMOTE (Synthetic Minority Oversampling Technique) to balance the dataset.

Hands-on Practice

• Loaded and explored multiple datasets.
• Cleaned missing and duplicate data.
• Performed exploratory data analysis.
• Encoded categorical features.
• Created new features.
• Balanced an imbalanced dataset using SMOTE.

Key Takeaways

• Data preprocessing is an essential step before building machine learning models.
• Clean data improves model performance.
• Data visualization helps understand patterns and detect anomalies.
• Feature engineering can improve predictive performance.
• Handling imbalanced data results in more reliable machine learning models.
